## 核心系统架构 (System Architecture)

### 1. 核心引擎层 (`core/`)
- **`renderer.py`**: 中央图像处理引擎。流程：加载 -> 智能缩放 -> 画布创建 -> 主题渲染 (渐变/磨砂/纸质纹理) -> 排版复合 -> 复合底色保存。
- **`typo_engine.py`**: 高精度排版引擎。处理字体度量、多行排版以及 **Radar 安全系统**（溢出检测）。
- **`metadata.py`**: 健壮的 EXIF/XMP 提取引擎，支持相机/胶片型号映射。

### 2. GUI 层 (`gui/`)
- **`panels/`**: 
    - **`border_panel.py`**: 边框处理中心，管理批量队列与实时预览。
    - **`contact_panel.py`**: 针对 135/120 高密度样片（Contact Sheet）的专项渲染。
- **`controllers/`**:
    - **`border_controller.py`**: **逻辑粘合层**。解耦 GUI 状态与核心渲染。负责将 UI 的“物理像素”输入换算为内核的“比例坐标” (参考基准: 4500px 长边)。
- **`components/`**: 可复用组件 (如 `SettingsGroup`, `ThumbnailStrip`)，采用变量观察模式。

### 3. 核心算法逻辑
- **像素比例系统**：UI 输入的所有 `px` 均为逻辑单位，以 **4500px 长边** 为 1:1 换算参考，实现跨分辨率渲染的一致性。
- **物理像素锚定**：所有边距（左/右/上/下）严格统一参考 **长边 (max(W,H))**，解决横竖构图缩放不一致问题。
- **曜石黑引擎**：采用垂直线性渐变叠加 **物理抖动 (Dithering)** 算法，通过高频随机噪点模拟磨砂纸张感，彻底消除 8-bit 色深下的渐变断层。
